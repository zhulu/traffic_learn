# 📝 实验记录：加密流量识别终极方案 (Stage 1 - 特征融合版)

## 1. 实验目标
在微观序列特征的基础上，引入 15s 流窗口内的宏观统计特征，旨在解决单一序列特征在类别区分度上的瓶颈，冲击 F1 > 0.9 的科研指标。

## 2. 特征工程：双轨并行 (Dual-Track Features)
* **轨道 A（微观序列）**：提取双向流前 32 个报文。采用“有符号长度” ($Length \times Direction$) 和 $\log(1+IAT)$ 构建 $32 \times 2$ 的时序张量。
* **轨道 B（宏观统计）**：提取 15s 窗口内全量报文的 10 维统计特征：
    * 长度统计：Mean, Std, Max, Min
    * 时间统计：Mean, Std, Max, Min
    * 吞吐量统计：Bytes/sec, Pkts/sec
* **预处理**：对宏观统计量进行 $\log(1+x)$ 归一化，防止吞吐量等大值对梯度造成冲击。

## 3. 模型架构：多模态融合网络 (Fusion Network)
* **时序分支**：1D-CNN 与 Transformer 并行提取。
* **统计分支**：直接输入统计向量。
* **融合逻辑**：在 MLP 决策层之前进行特征拼接 (`Concat`)。
    * `Fused_Vector = [CNN_feat, Transformer_feat, Global_Stats]`

## 4. 实验结果分析 (Final Results)
本次实验成功突破了基准方案的性能天花板：

| 指标 (Metric) | 优化前 (Baseline) | 特征融合优化后 | 增幅 |
| :--- | :--- | :--- | :--- |
| **准确率 (Accuracy)** | 0.8633 | **0.9330** | **+7%** 🚀 |
| **宏平均 F1 (Macro F1)** | 0.7864 | **0.8965** | **+11%** 🚀 |
| **VPN 精确率 (Precision)** | 0.76 | **0.94** | **极大幅度提升** |
| **VPN 召回率 (Recall)** | 0.58 | **0.75** | **稳健增长** |

### 💡 导师总结
引入宏观统计量是本次成功的关键。模型现在既能通过 CNN 看到“握手”的微观指纹，又能通过统计量看到“吞吐”的宏观轮廓。VPN 识别的 F1 分数达到 0.83，整体 F1 逼近 0.9，已经完全具备了作为 **Scenario A 第一阶段“守门员”** 的资格。
