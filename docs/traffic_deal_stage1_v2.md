# 📝 实验记录：加密流量识别优化方案 (Stage 1 - 优化迭代一)

## 1. 实验背景与优化动机
在基准实验 (Baseline) 中，虽然模型整体准确率达到了 0.86，但**VPN 的召回率 (Recall) 仅为 0.5821**。由于数据集存在严重的类别不平衡（Non-VPN 样本约为 VPN 的 3.4 倍），模型陷入了“偏袒多数类”的局部最优解。
为此，我们从**特征工程**和**代价敏感学习**两个维度进行了深度优化。

## 2. 特征工程优化 (Feature Engineering)
[cite_start]原论文证明了时间相关特征（如 IAT）在加密流量分类中的核心地位 [cite: 82-84, 87-89]。为更充分地发挥深度网络的能力，我们对特征输入层进行了重构：

* **融合构建“有符号长度” (Signed Length)**：
  * **做法**：将原本独立的“包长度 (Length)”和“方向 (Direction, 0/1)”融合。公式为：$Length \times (2 \times Direction - 1)$。
  * **意义**：前向报文长度为正，后向报文长度为负。这一操作不仅将特征维度从 3 维降维至 2 维（提升特征密度），还能让 1D-CNN 分支通过正负数值的交替震荡，更敏锐地捕捉到 TCP/UDP 的“请求-响应”交互节奏。
* **时间特征对数化平滑**：
  * **做法**：应用 $\log(1 + IAT)$ 对到达间隔时间进行缩放。
  * **意义**：缓解网络延迟特征的长尾分布现象，帮助 Transformer 的自注意力机制更稳定地聚焦于应用层真实的心跳节拍，对抗 VPN 隧道的封装扰动。

## 3. 训练策略优化 (Training Strategy)
针对数据集极度不平衡的痛点，我们在损失函数层面引入了干预机制：

* **动态代价敏感学习 (Cost-Sensitive Learning)**：
  * **做法**：在 `BCEWithLogitsLoss` 中引入 `pos_weight` 参数。通过动态计算训练集中多数类与少数类的比例（$N_{nonvpn} / N_{vpn} \approx 3.42$），赋予 VPN 样本约 3.4 倍的错误惩罚权重。
  * **意义**：强迫模型在梯度更新时，对误判真正的 VPN 隧道付出更“高昂”的代价，从而从根本上扭转模型保守的预测倾向，提升召回率。

## 4. 模型架构认知对齐 (Architecture Clarification)
明确了当前模型的拓扑结构为 **双分支并行 (Parallel Branches)** 而非串行：
* **分支 A (1D-CNN)**：提取局部包长度的突发模式。
* [cite_start]**分支 B (Transformer)**：基于 15s 截断窗口 [cite: 207-209]，捕捉全局序列的时间依赖关系。
* **融合与输出**：两者提取的特征向量进行 `Concat` 拼接，输入后端 MLP，最终通过单一神经元（Logit）结合 Sigmoid 进行二分类判别，而非 Softmax。

## 5. 优化成果对比 (Results Comparison)
经过上述优化，模型在验证集上的表现产生了关键性突破：

| 指标 (Metric) | 基准 (Baseline) | 优化后 (Optimized) | 变化 |
| :--- | :--- | :--- | :--- |
| **准确率 (Accuracy)** | 0.8633 | **0.8542** | 轻微下降 (Trade-off) |
| **VPN 召回率 (Recall)** | 0.5821 | **0.8383** | **显著提升 (+44%)** 🚀 |
| **宏平均 F1 (Macro F1)** | 0.7864 | **0.8118** | **整体增强 (+3%)** |

### 💡 结论与下一步计划
优化极为成功！通过**特征融合**与**加权损失**，我们成功突破了类别不平衡的瓶颈，使模型具备了高敏感度的 VPN 隧道嗅探能力（抓出了 84% 的隐蔽隧道）。尽管牺牲了部分 Precision（错杀率上升），但在网络安全的工程实践中，“高召回、低漏报”是第一诉求。
